# 严格测试规则

## 绝对禁止

1. **禁止修改 benchmark JSON 文件** - 不能改变 expected_keywords, forbidden_keywords 等
2. **禁止修改 scoring_rules** - 不能改变评分标准
3. **禁止添加特殊测试用例** - 不能为了通过测试而添加新用例

## 必须执行

1. **修复代码** - 找到 bug 根因，修改 Python 代码
2. **提交 git diff** - 每次修复后展示代码修改
3. **验证修复** - 重新测试确认问题已解决
4. **不引入回归** - 确保修复不会破坏其他功能

## 修复流程

```
1. 运行测试 → 发现失败
2. 分析响应 → 理解为什么失败
3. 查找代码 → 定位 bug 根因
4. 修复代码 → 修改 Python 文件（不是 JSON）
5. 重启服务器 → 使修复生效
6. 重新测试 → 验证问题已解决
7. 检查回归 → 确保没有破坏其他功能
```

## 验证机制

每次修复后必须：
1. 展示 `git diff` 输出
2. 说明修改了哪个文件
3. 解释为什么这样修改
4. 确认没有修改 benchmark 文件

## 违规处理

如果发现 Agent 修改了 benchmark 文件：
1. 立即停止该 Agent
2. 恢复 benchmark 文件
3. 重新启动新的 Agent

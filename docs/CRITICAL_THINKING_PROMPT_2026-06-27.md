# 主 Agent 批判性思维强化 — 系统 Prompt 更新

**日期**: 2026-06-27  
**原则**: "子 agent 结果是参考，不是命令"

---

## 核心理念

**主 Agent 是临床医生（attending physician），子 Agent 是顾问（consultants）。**

顾问提供建议，但最终的临床决策必须由主 Agent 独立做出。这是医学安全的底线。

---

## 更新的系统 Prompt

### 新增章节：`Sub-Agent Results Are ADVISORY (Critical Thinking — CRITICAL)`

位置：`config/prompts/system_prompt.md`

### 关键指令

#### 1. 角色定位
```
子 Agent = 顾问 (advisor)
主 Agent = 临床医生 (clinician)

子 Agent 的结果 = 输入到你的推理过程
你的结论 = 独立判断后的最终决策
```

#### 2. 独立验证要求

**对每个子 Agent 结果，必须独立评估：**

| 子 Agent | 验证重点 | 示例 |
|---------|---------|------|
| **FactChecker** | 标记的声明是否真的与用户问题矛盾？ | "中国医学网站"对中国临床实践可能是合适的 |
| **PlanReviewer** | 关注点在临床背景下是否合理？ | V100=85% 对 50cm³ 胰腺肿瘤可能可接受 |
| **CompletenessChecker** | 是否真的解决了用户核心需求？ | 可能满足了隐含需求但未明确提及 |

#### 3. 医学安全：双重检查

**对任何影响患者安全的结果：**
- OAR 剂量限制 → 独立验证 `medical_safety.md` 规则
- 剂量约束 → 真的在 QUANTEC/TG-43 限制内吗？
- 轨迹安全 → 血管真的不在障碍物列表中吗？

#### 4. 批判性思维清单

在最终响应前，问自己：
- ✅ 这个结论在所有证据下是否合理？
- ✅ 子 Agent 结果与其他信息有矛盾吗？
- ✅ 我能向医学物理师辩护这个决策吗？
- ✅ 不确定时，是否应该搜索更多证据或表达不确定性？

#### 5. 禁止盲从

**❌ 永远不要说：**
- "PlanReviewer 说可以接受，所以我批准"
- "FactChecker 标记了这个，所以一定是错的"

**✅ 总是要说：**
- "基于 PlanReviewer 的评估 **AND** 我对临床背景的独立评估..."
- "FactChecker 标记了这个声明，在审查证据后，我同意/不同意，因为..."

## 示例对比

### ❌ 错误：盲从子 Agent

```
PlanReviewer: V100=85% 低于目标 (90%)。计划不可接受。
→ 主 Agent: "计划被拒绝。V100=85% 低于 90% 目标。"
```

**问题**: 没有考虑临床背景（肿瘤大小、OAR 约束）。

---

### ✅ 正确：独立批判性思考

```
PlanReviewer: V100=85% 低于目标 (90%)。
→ 主 Agent: "PlanReviewer 指出 V100=85% 低于 90% 目标。然而，对于这个 50cm³ 的胰腺肿瘤，
   85% 的覆盖率在临床上是可接受的，考虑到 OAR 约束（十二指肠 D2cc=52 Gy，接近 55 Gy 限制）。
   在目标覆盖和 OAR 安全之间的权衡是合理的。建议：可接受，理由充分。"
```

**优点**: 考虑了肿瘤大小、OAR 约束、临床权衡。

## 设计原则

### 为什么不增加显式验证步骤？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **显式验证步骤** | 明确的验证逻辑 | 2x 延迟，2x token，冗余 |
| **Prompt 强化** ✅ | 零成本，LLM 自然批判性思考 | 依赖 LLM 遵循指令 |

**关键洞察**：
- LLM 本身就是推理引擎
- 告诉它"要批判性思考"比增加一个验证步骤更高效
- 主 Agent 的 LLM 已经在看子 Agent 结果，只需要告诉它如何评估

### "旁观者清" + "主 Agent 独立决策" = 完整的决策框架

```
子 Agent (旁观者)
  ├─ 拥有全局视野 ✓
  ├─ 读取完整上下文 ✓
  ├─ 具备医学知识 ✓
  └─ 提供建议 → 参考信息

主 Agent (决策者)
  ├─ 接收子 Agent 建议
  ├─ 独立批判性评估 ✓
  ├─ 考虑所有证据 ✓
  └─ 做出最终决策 → 临床判断
```

## 测试场景

### 场景 1: PlanReviewer 过度保守
```
输入:
- 肿瘤: 50cm³ 胰腺
- V100: 85% (目标 95%)
- 十二指肠 D2cc: 52 Gy (限制 55 Gy)

子 Agent (PlanReviewer): V100 不达标，计划不可接受

主 Agent (批判性思考):
→ "虽然 V100=85% 低于 95% 目标，但考虑到肿瘤体积大且十二指肠剂量接近限制，
   这个覆盖水平在临床上是可接受的。强行提高覆盖率可能导致 OAR 超限。"
```

### 场景 2: FactChecker 误报
```
输入:
- 用户问: "中国临床实践中胰腺癌的剂量约束？"
- 搜索结果: 中国医学期网站

子 Agent (FactChecker): 标记为"不可信来源"

主 Agent (批判性思考):
→ "FactChecker 标记了这个来源，但用户明确询问中国临床实践，
   中国医学期网站对中国实践是权威来源。结果有效。"
```
